What is Azure Kubernetes Service?

Azure Kubernetes Service (AKS) is a fully managed Kubernetes container orchestration service provided by Microsoft Azure. It simplifies the deployment, management, and scaling of containerized applications using Kubernetes, without requiring you to manage the underlying Kubernetes control plane infrastructure.

AKS handles critical tasks such as health monitoring, automatic upgrades, and scaling, allowing development teams to focus on building applications rather than managing infrastructure.

Key Features

  • Managed Control Plane: Azure manages the Kubernetes master nodes at no extra cost, handling upgrades, patches, and high availability.
  • Automatic Scaling: Supports both Horizontal Pod Autoscaler (HPA) and Cluster Autoscaler to dynamically adjust resources based on demand.
  • Integrated Developer Tools: Deep integration with Azure DevOps, GitHub Actions, Azure Container Registry, and Visual Studio Code.
  • Azure Monitor Integration: Built-in monitoring and logging through Azure Monitor and Container Insights for full observability.
  • Security & Identity: Integrates with Microsoft Entra ID (Azure AD) for role-based access control (RBAC) and supports Azure Policy for governance.
  • Virtual Nodes: Burst workloads onto Azure Container Instances using virtual nodes for near-instant pod scaling.
  • Multiple Node Pools: Run different VM sizes and OS types (Linux and Windows) in the same cluster using multiple node pools.
  • GPU Support: Schedule GPU-enabled workloads for AI/ML inference and compute-intensive tasks.

Core Concepts

  • Cluster: A set of nodes (virtual machines) that run containerized applications. In AKS, the control plane is managed by Azure.
  • Node Pool: A group of nodes within a cluster that share the same configuration (VM size, OS, Kubernetes version).
  • Pod: The smallest deployable unit in Kubernetes, containing one or more containers that share storage and networking.
  • Deployment: A Kubernetes object that manages a replicated application, ensuring the desired number of pod replicas are running.
  • Service: An abstraction that exposes a set of pods as a network service, with options for ClusterIP, NodePort, and LoadBalancer types.
  • Namespace: A virtual cluster within a Kubernetes cluster, used to organize and isolate resources across teams or environments.
  • Ingress: An API object that manages external HTTP/HTTPS access to services within a cluster, typically via Azure Application Gateway or NGINX.

Common Use Cases

  • Microservices architectures requiring independent scaling and deployment of services
  • CI/CD pipelines for containerized applications
  • Batch processing and data pipeline workloads
  • Machine learning model serving and inference at scale
  • Hybrid and multi-cloud application deployments using Azure Arc
  • Lift-and-shift of on-premises containerized workloads to the cloud

Pricing Model

AKS has no charge for the managed Kubernetes control plane. You pay only for the virtual machine instances, associated storage, and networking resources used by your agent nodes. Additional costs may apply for optional features such as:

  • Uptime SLA (for production-grade availability guarantee)
  • Azure Monitor and Container Insights
  • Azure Container Registry for storing container images
  • Load balancer and public IP addresses
